# Sub-Proxy 优化建议文档（面向对外使用）

本文档用于梳理当前项目在“个人可用”到“可对外稳定交付”之间仍建议补强的方向，并给出可执行的优化清单。

## 1. 总体优先级（建议按顺序推进）

### P0（必须）

- 安全基线完善：默认密钥、默认账号、鉴权边界、输入校验、CORS 与下载接口策略。
- 数据一致性与可恢复：写入原子性、备份策略、恢复演练、升级兼容。
- 错误可观测：结构化日志、请求链路 ID、关键接口错误码规范。
- 关键流程测试：订阅生成、rule-providers、流量/有效期限制、导入导出。

### P1（强烈建议）

- 运营能力：版本升级引导、配置迁移、系统健康检查增强。
- 性能优化：大 YAML 生成性能、前端大体积包拆分、节点拉取并发控制。
- 可维护性：配置与常量分层、接口契约文档化、模块职责边界清晰化。

### P2（体验增强）

- 多租户/角色权限、审计日志、操作回滚。
- 规则市场（预设规则模板中心）、一键导入社区规则。
- 可视化监控面板（流量趋势、失败率、拉取成功率）。

## 2. 安全与合规优化

### 2.1 鉴权与凭据

- 强制首次登录改密，禁止默认密码长期生效。
- JWT 密钥必须由环境变量注入，启动时校验强度与长度。
- 管理端接口统一鉴权，公开接口仅保留必要的 subscribe 下载能力。
- 预览接口与管理接口分离，避免误暴露内部数据。

### 2.2 输入与输出安全

- 对 visualConfig、ruleProviders、secondHopConfig 做更细粒度 schema 校验（不仅 any）。
- 所有文本输入统一长度限制、字符白名单或风险字符过滤。
- 防 SSRF：对第三方规则/订阅 URL 做协议与目标地址限制（禁止内网/回环）。
- 下载响应头中避免暴露不必要的内部信息。

### 2.3 运行环境安全

- 提供生产推荐配置模板（反向代理、HTTPS、最小权限挂载）。
- 容器运行用户降权（非 root）、文件系统只读化（可写目录白名单）。
- 上传目录与数据目录分离，限制可执行权限。

## 3. 稳定性与数据可靠性

### 3.1 数据层

- lowdb 写入增加原子保障与异常回滚策略。
- 增加数据版本号与迁移脚本（db schema migration）。
- 导入前做完整校验与 dry-run，导入后自动快照备份。

### 3.2 关键业务流程

- 订阅生成失败时提供可读错误定位（哪一段配置失败）。
- 对第三方规则与节点拉取增加失败重试、熔断、退避。
- rule-providers 为空或不匹配时，给出明确告警而非静默失败。
- 预览与真实下载逻辑尽量复用同一生成管线，避免“预览正常、下载异常”。

## 4. 性能优化

### 4.1 后端

- 大 YAML 场景优化字符串拼接策略，减少重复序列化。
- 节点拉取采用并发上限控制（例如 p-limit），避免阻塞主请求。
- 增加短时缓存（仅对可缓存部分），减少重复生成压力。

### 4.2 前端

- 当前产物体积偏大，建议按页面与编辑器进行动态导入（Monaco 懒加载）。
- 重型弹窗（配置预览）可拆分组件并按需加载。
- 列表页可引入虚拟滚动（当订阅数量较大时）。

## 5. 可观测性与运维

### 5.1 日志

- 统一结构化日志格式（JSON），区分 info/warn/error。
- 关键字段：requestId、userId、subscriptionId、duration、statusCode。
- 敏感字段脱敏（URL 凭据、token、密码相关内容）。

### 5.2 监控指标

- 订阅生成成功率、平均耗时、失败原因分布。
- 第三方规则拉取成功率、节点拉取失败率。
- 流量统计写入失败次数、数据库写入失败次数。

### 5.3 健康检查

- /health 增强为分级健康（应用、数据文件可写、磁盘空间）。
- 增加 readiness/liveness 语义，方便容器编排。

## 6. 测试与质量保障

### 6.1 自动化测试建议

- 单元测试：yamlGenerator（含 rule-providers、chainProxies、默认规则）。
- 集成测试：订阅创建->编辑->预览->下载全链路。
- 回归测试：流量耗尽、过期限制、rule-providers 引用去重。

### 6.2 工程质量

- 增加 lint/typecheck/test 的统一脚本与 CI 流程。
- PR 门禁：类型检查 + 构建 + 关键测试集必须通过。
- 约束“不可回退行为”用测试固化（例如 Form.List key spread 警告类问题）。

## 7. 产品化能力（给别人用必须考虑）

### 7.1 多环境与配置管理

- 提供 dev/staging/prod 配置模板与说明。
- 环境变量文档化（必填、可选、默认值、风险项）。
- 一键部署脚本加参数校验，避免误配置上线。

### 7.2 可用性与易用性

- 首次安装向导（改密、基础 URL、默认订阅模板）。
- 错误提示标准化（可读错误 + 建议操作）。
- 配置预览支持“差异对比”（当前 vs 上次保存）。

### 7.3 文档与支持

- 提供管理员手册与用户手册拆分版本。
- FAQ 聚焦真实高频问题（CORS、规则不生效、订阅客户端兼容差异）。
- 发布记录（Release Note）说明破坏性变更与迁移步骤。

## 8. 建议里程碑（4 周示例）

### 第 1 周：安全与稳定基线

- 默认密码与密钥策略、输入校验补强、导入导出校验、日志结构化。

### 第 2 周：质量体系

- 建立测试基线（生成器单测 + 核心接口集成测试）与 CI 门禁。

### 第 3 周：性能与体验

- Monaco 懒加载、列表与预览性能优化、节点拉取并发治理。

### 第 4 周：产品化收尾

- 环境配置文档、运维手册、版本发布说明、升级迁移脚本。

## 9. 当前项目可立即落地的 10 条行动项

- 增加 `pnpm lint`、`pnpm typecheck`、`pnpm test` 脚本并接入 CI。
- 为 `visualConfig` 与 `ruleProviders` 增加严格 zod 校验。
- 统一处理第三方 URL 的 SSRF 风险与协议限制。
- 为 `/api/subscription/*preview*` 增加频率限制与审计日志。
- 将 YAML 生成逻辑封装为单一服务，预览/下载共用。
- 引入 requestId 中间件，日志全链路可追踪。
- 增加 db schema version + migration 执行器。
- 提供生产环境 `.env.example` 与安全配置检查启动项。
- 前端将 Monaco 编辑器改为动态加载并拆分 chunk。
- 增加“规则生效诊断”页面（rule-provider 拉取状态 + 解析状态）。
